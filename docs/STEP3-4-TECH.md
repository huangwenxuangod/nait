# Step 3 · 准备页：个性化准备确认系统

## 设计目标

把传统"静态清单"升级为**个性化准备确认系统**，让每一项物料都不是"看就完了"，而是被用户主动确认，并为后续制作页的智能调整提供依据。

## 用户行为

进入准备页后，AI 已经根据 Step 1 抓取的教程内容自动生成了该款式的 BOM 清单（Bill of Materials），分两段：

| 分类 | 内容举例 |
|------|----------|
| **基础工具** | 底胶、烤灯、封层、搓条、清洁液等 |
| **专属色号** | 该款式用到的具体甲胶颜色、亮片、镭射粉等 |

用户逐一确认自己是否拥有每项物料。

## 核心交互

### 每项物料的两种选择

```
┌──────────────────────────────────────────┐
│  底胶 Base Coat            ✅ 我有  ❌ 我没有  │
│  持久不脱落                              │
└──────────────────────────────────────────┘
```

- 两个按钮并排，用户在每一项上**必须做出选择**（不能跳过）。
- `✅ 我有`：绿色高亮，计入已备齐计数。
- `❌ 我没有`：玫瑰金色高亮，触发替代方案面板。

### 替代方案面板

点击 `❌ 我没有` 后，该项下方展开替代方案：

```
┌──────────────────────────────────────────┐
│  UV / LED 烤灯            ✅ 我有  ❌ 我没有  │
│  60s~90s 固化                            │
│  ┌─ 💡 替代方案 ─────────────────────┐   │
│  │  🔧 LED手电筒 + 锡纸聚光照射       │   │
│  │  🔧 户外阳光直射 2-3 分钟（晴天）   │   │
│  └───────────────────────────────────┘   │
└──────────────────────────────────────────┘
```

- 色号类替代：显示相近色号代码 + 搭配方案，用 `🎨` 图标区分。
- 工具类替代：显示日常物品替代方案，用 `🔧` 图标区分。
- 替代面板可折叠/展开，不占用过多空间。

### 顶部状态计数

```
已备齐 4 项 · 需替代 2 项
```

- 实时更新，"备齐"为 mint-green，"需替代"为 rose。
- 所有项确认完毕后才出现完整计数；未全部确认时显示"已确认 X / Y · 余下的请选择"。

### CTA 解锁逻辑

| 条件 | 按钮状态 | 文案 |
|------|----------|------|
| 未全部确认 | 灰色禁用 | 请确认每一项物料 |
| 全部确认，有缺失 | 渐变色激活 | 已记录替代方案 · 开始制作 |
| 全部确认，无缺失 | 渐变色激活 | 全部就绪 · 开始沉浸式制作 |

**设计意图**：强化参与感——用户不是看一眼清单，而是有"我做完了准备"的心理锚点。

### 数据传递

所有标记为 `❌ 我没有` 的物料 ID 组成 `缺失清单`，在点击"开始制作"时传入 Step 4 制作页，用于：

- 制作页顶部显示"替代方案已就绪"提醒
- 在相关步骤中给出针对性的操作建议（如缺了烤灯，会在照灯步骤提示户外阳光替代法）

# Step 4 · 制作页：AI 辅助多步骤执行系统

## 设计目标

让用户在**暗黑沉浸式环境**中，跟着 AI 拆解后的步骤**一步步执行**，每一步都有：
- 精确到秒的计时
- AI 翻译后的操作指导
- 完成后 AI 拍照核验
- 最终一键生成社交分享视频

## 页面结构

```
┌────────────────────────────────────┐
│  ⚠️ 替代方案已就绪 · 2 项物料需注意    │  ← 折叠条（来自 Step 3）
├────────────────────────────────────┤
│  · LIVE SOP          Step 04 / 06  │  ← 顶部状态栏
├────────────────────────────────────┤
│                                    │
│        [ 当前步骤图片 · 45vh ]       │
│                                    │
├────────────────────────────────────┤
│  Step 04 · 上色胶                   │  ← 步骤标题
│  刷头蘸取少量色胶薄涂两层            │  ← 主指令（大字）
│  第一层半透明即可，照灯60s后...      │  ← 详细说明
├────────────────────────────────────┤
│  ┌ AI 提示小气泡 ─────────────────┐ │
│  │ 📹 原视频说：「上两遍颜色...」  │ │  ← 原文
│  │ ─────────────────────────── │ │
│  │ ✨ AI帮你翻译：第一层薄涂...   │ │  ← AI 解读
│  │              💡 长按看原视频  │ │
│  └─────────────────────────────┘ │
├────────────────────────────────────┤
│                                    │
│  ⭕ 90s 🔆 烤灯固化中    🔊 语音已唤醒  │  ← 计时 + 语音助手
│                     📸 拍照核验      │
│                                    │
│  ← 上一步                      下一步 →│  ← 步骤导航
│              长按退出                │
└────────────────────────────────────┘
```

## 六步 SOP 流程

| Step | 标题 | 时长 | 烤灯 | 说明 |
|------|------|------|------|------|
| 01 | 修形打磨 | 120s | ✗ | 海绵锉单向修甲型，不能来回拉锯 |
| 02 | 上底胶 | 60s | ✗ | 刮掉 2/3 胶再上，两侧留 0.5mm |
| 03 | 照灯固化底板 | 90s | 🔆 | 手平放灯内不要动 |
| 04 | 上色胶 | 120s | ✗ | 第一层薄涂，第二层厚涂显色 |
| 05 | 照灯 + 封层 | 120s | 🔆 | 先照 90s 再封层再照 120s |
| 06 | 精修检查 | 60s | ✗ | 擦浮胶 + 涂营养油 |

每步的时长是**根据该步骤实际所需时间独立设置的**，不是统一倒计时。

## 烤灯环节特殊处理

涉及照灯固化的步骤（03、05）在 UI 上有明确区分：

| 元素 | 普通步骤 | 烤灯步骤 |
|------|----------|----------|
| 倒计时环颜色 | 玫瑰金 `#D4A3A3` | 琥珀色 `#F5C542` |
| 计时器标签 | "固化中" | "🔆 烤灯固化中" |
| 心理暗示 | 正常节奏 | 提示用户不要中断，严格计时 |

**设计意图**：烤灯固化是美甲中容错率最低的环节，时间短了胶不干，时间长了胶变黄。用颜色和标签双重提醒用户这步不能马虎。

## AI 提示小气泡

每一步在主指令下方嵌入一个半透明卡片，用**上下结构**对比展示：

```
📹 原视频说
「照灯烤一下，干了就行」

────────────────

✨ AI 帮你翻译
UV/LED灯固化至少90秒，手平放灯内不要动。
⚠️ 未完全固化会导致甲面变软起皱

                     💡 长按查看原视频片段
```

- `📹 原视频说`：保留教程原始口吻，用 rose 色标识。
- `✨ AI 帮你翻译`：把模糊、主观的原始说法翻译为**可操作的具体指令**，用 mint-green 色标识。
- 长按气泡 0.6 秒弹出深色蒙层，模拟原视频片段播放。

**设计意图**：评审可以一眼看到「原文 vs 翻译」的差异，且这个对比不占额外页面，直接嵌入在执行步骤中。

## 拍照核验系统

### 触发时机

每步倒计时归零后，底部 `📸 拍照核验` 按钮由灰色变为高亮，提示用户拍照。

### 核验流程

```
倒计时归零
  → 用户点击"拍照核验"
    → [2s AI 分析动画]
      → ✅ 核验通过 → "做的很棒!" → 进入下一步
      → ❌ 检测到问题 → 显示具体建议 → 调整后重试
```

### 不合格时的反馈示例

```
┌──────────────────────────────────────────┐
│  ✕  检测到需要调整                       │
│  甲面边缘检测到溢胶，请用棉签蘸酒精      │
│  从甲缘外侧向内擦拭                      │
│                                          │
│        [调整后重试]    [重新拍照]         │
└──────────────────────────────────────────┘
```

- 不合格提示是**具体可操作的建议**，不是笼统的"不合格"。
- 提供一个随机的问题库，覆盖常见美甲失误：溢胶、颜色不均、气泡、包边不完整。
- 用户调整后点"调整后重试"→ 重新进入当前步骤的 active 倒计时状态。

**设计意图**：不是让用户反复拍照碰运气，而是给出能立刻执行的改进建议，缩短"发现问题 → 解决问题"的路径。

## 步骤导航

- 提供 `← 上一步` / `跳过核验 →`（或 `下一步 →`）按钮。
- "跳过核验"用于用户自信无需 AI 检查的场景，保留灵活性。
- 顶部始终显示 `Step 0X / 06`，让用户知道进度。

## 语音助手

底部常驻音频波形动画 + "语音已唤醒"标签。

**设计意图**：制作过程中双手被占用，语音助手让用户无需触碰屏幕即可获取指令。当前为 UI 占位，等接入真实 TTS。

## 缺失物料提醒

从 Step 3 传入的缺失物料信息在页面顶部以 amber 色提示条展示：

```
⚠️ 替代方案已就绪 · 2 项物料需注意   [收起]
```

点击可展开查看完整替代清单，展开后自动调整页面 padding 避免遮挡内容。

**设计意图**：在用户执行具体步骤时，如果遇到缺某个工具/色号的情况，不需要退回准备页，制作页顶部就有替代方案提醒。

## 完成 & 视频生成

### 最后一步核验通过后

```
┌──────────────────────────────────────────┐
│                                          │
│           ✓  全部步骤完成！              │
│    恭喜你完成了第一款 AI 辅助美甲        │
│        现在可以生成你的 DIY 视频了       │
│                                          │
│        [🎬 生成我的 DIY 视频]            │
│          返回首页                        │
└──────────────────────────────────────────┘
```

### 视频生成流程

```
点击"生成我的 DIY 视频"
  → [4s AI 剪辑动画]
  → 视频预览页（9:16 竖屏，抖音格式）
    → 展示话题标签：#美甲DIY #指尖SOP #沉浸式
    → [一键转发到抖音]（红色按钮，匹配抖音品牌色）
    → [稍后再说]
```

**设计意图**：从"看教程" → "做出来" → "分享出去"，形成完整的用户闭环。视频自动剪辑用户在制作过程中的多角度照片/视频片段，配上字幕和音乐，降低用户制作社交内容的门槛。

## Step 3 → Step 4 数据流

```
Step 3 准备页
  │
  ├── 用户确认每项物料（我有/我没有）
  ├── 缺失项收集为 missingItems[]
  │
  └── onClick"开始制作" ──────────────────→ Step 4 制作页
                                               │
                                               ├── 顶部替代方案提醒
                                               ├── 6 步 SOP 逐一执行
                                               ├── 每步 AI 提示 + 烤灯区分
                                               ├── 拍照核验（合格/不合格 + 建议）
                                               └── 完成 → 生成抖音视频 → 一键转发
```
